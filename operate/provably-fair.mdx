Every Origami round is generated from verifiable randomness, not hidden server logic. The outcome is derived from a deterministic seed, signed with a server key, and accompanied by a proof that anyone can verify against our public key.

The ECVRF RNG that powers every game is independently certified by **Gaming Laboratories International (GLI)**.

## How a round is built

<Steps>
  <Step title="Seed">
    Before the round resolves, we compose a seed from five values:

    | Component | Source | Purpose |
    |:--|:--|:--|
    | `gameId` | System | Which game is playing |
    | `tenantId` | Operator | Your platform identifier |
    | `tenantUserId` | Operator | Player ID on your platform |
    | `nonce` | System | Incrementing counter, one per round per player |
    | `clientSeed` | Player | Player-provided entropy (can be changed anytime) |

    The seed is a string formatted as `gameId-tenantId-tenantUserId-nonce-clientSeed`. Same inputs always produce the same random stream and outcome.
  </Step>
  <Step title="Proof">
    The server runs the seed through an Elliptic Curve Verifiable Random Function (ECVRF) using its private key. ECVRF produces a deterministic random output and a cryptographic `proofString`. We follow the IETF spec `draft-irtf-cfrg-vrf-15`.
  </Step>
  <Step title="Outcome">
    The proof's hash seeds a deterministic byte generator. Each game transforms those bytes into its outcome using unbiased logic: a bucket in Plinko, a card draw in Blackjack, a target comparison in Limbo.
  </Step>
  <Step title="Verification">
    After the round, the player receives the inputs (including the `nonce` and `clientSeed`), the `proofString`, and our public key. Anyone can run ECVRF verification to confirm the outcome was generated from those exact inputs and was not tampered with.
  </Step>
</Steps>

## Why it matters

**For operators**
- Trust beyond traditional RNG certification. Players don't have to take your word, or ours.
- Auditable trail for every round, signed and timestamped.
- The `nonce` increments on every completed round. Players can confirm no rounds were skipped or inserted.
- GLI certification for regulated-market requirements.

**For players**
- No one, not you or us, can predict or alter a result after the seed is committed.
- Players can change their `clientSeed` at any time to introduce their own entropy.
- Any past round can be re-verified later with the public key alone.

## Under the hood

The TypeScript SDK wraps a Rust implementation (`fastcrypto` via NAPI-RS) for ECVRF operations. Two packages are published on npm:

| Package | Version | Purpose |
|:--|:--|:--|
| `@betorigami/ecvrf` | `1.0.2` | Low-level ECVRF primitives with prebuilt native bindings for macOS, Linux, Windows, Android, and FreeBSD |
| `@betorigami/provably-fair-rng` | `1.0.0` | Origami-specific seed composition, generation, and verification built on `@betorigami/ecvrf` |

Our private key never leaves secure servers. The public key is safe to expose to any client for independent verification.

## GLI certification

The ECVRF RNG has passed Gaming Laboratories International's randomness evaluation.

| | |
|:--|:--|
| **Report** | RN-246-OR2-25-02 |
| **Report date** | 10 October 2025 |
| **Jurisdiction** | UK Remote |
| **Standards** | Remote Gambling and Software Technical Standards (January 2025) |
| **Issuing lab** | GLI Europe B.V. (Haarlem, Netherlands) |
| **Evaluating labs** | GLI LLC (Lakewood, NJ, USA) and GLI Australia |
| **Scope** | Software verification, source code review, and statistical analysis of `@betorigami/ecvrf` v1.0.2 and `@betorigami/provably-fair-rng` v1.0.0 |
| **Result** | Pass at 95%, 98%, and 99% confidence levels |

The evaluation tested the RNG against parameter sets for the following games:

| Game | Range | Draws per round | Draws tested |
|:--|:--|:--|:--|
| Dice | 0–10,000 | 1 | 2,000,000,000 |
| Limbo | 1–1,000,000 | 1 | 1,000,000,000 |
| Keno | 0–39 | 10, no replacement | 50,000,000 |
| Baccarat | 0–51 | 6, with replacement | 50,000,000 |
| Mines | 0–24 | Up to 24, no replacement | 20,000,000 |
| Diamonds | 0–6 | 5, with replacement | 5,000,000 |
| Plinko | 0–1 | 8–16, with replacement | 1,000,000 |
| Roulette | 0–37 | 1 | Statistical battery |
| Wheel | 0–49 | 1 | Statistical battery |

The RNG passed the full test battery (Runs, Serial Correlation, Interplay Correlation, Adjacency Max-Min, Adjacency High-Low, Adjacency Blocks, Coupon Collector's, Duplicates, Overlaps, Permutation, Total Distribution, Count of Counts, and DieHard) at all three confidence levels.

Games with RNG parameters outside the scope of this report require a separate evaluation before use in regulated markets.
